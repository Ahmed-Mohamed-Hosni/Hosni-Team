# WBS Dictionary – GCHA Phase 1

This document defines selected Work Packages for the GCHA project and clearly specifies the Definition of Done (DoD) for each package to prevent the "90% Complete Syndrome".

---

## Berlin Site

**ID:** BER-001  
**Title:** Patient Registration Module  

**Description of Work:**  
Develop a patient registration interface that allows hospital staff to enter and manage patient demographic information. The module will securely store patient records in the centralized hospital database.

**Definition of Done (DoD):**
1. Code successfully passes peer review.
2. Interface tested with sample patient data.
3. Patient information correctly stored in the database.
4. Technical documentation uploaded to GitHub.

---

## Cairo Site

**ID:** CAI-001  
**Title:** Appointment Scheduling System  

**Description of Work:**  
Develop a scheduling system that allows hospital staff to create, modify, and cancel patient appointments. The system must synchronize with the hospital's central scheduling database.

**Definition of Done (DoD):**
1. Scheduling features tested and working correctly.
2. Appointment data successfully synchronized with the central database.
3. User interface reviewed and approved by stakeholders.
4. Documentation uploaded to the Git repository.

---

## Bangalore Site

**ID:** BAN-001  
**Title:** Nurse Login Screen  

**Description of Work:**  
Develop a secure login screen for nurses to access the hospital system. The system must authenticate users and provide access based on assigned roles.

**Definition of Done (DoD):**
1. Authentication functionality implemented and tested.
2. Code passes peer review.
3. User interface matches the approved design specifications.
4. Documentation uploaded to GitHub.
